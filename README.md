# CRM SkyBud — B2B Wärmepumpe Рекламация

Інтерактивний плейбук CRM-воронки **«Рекламація»** (Warranty Claims / Beschwerdemanagement)
для SkyBud — повний цикл обробки скарг на монтаж теплових насосів у сегменті B2B.

🔗 **Демо:** https://lysakanatoliia-png.github.io/B2B-W-rmepumpe-Montage-/

## Що це

Візуальна документація бізнес-процесу: від отримання заявки клієнта до усунення проблеми,
розрахунку фактичної собівартості та закриття картки. Дашборд показує етапи воронки,
відповідальних, дії по ролях, CRM чек-листи та критерії завершення кожного етапу.

## Структура воронки (v2.1)

7 етапів і 6 ролей:

| Етап | Назва | Відповідальний |
|------|-------|----------------|
| **R1** | Нова рекламація | CRM (автоматично) |
| **R2** | Технічний аналіз і рішення | Montageleiter |
| **R2.1** | Розрахунок вартості матеріалів | Інженер-ассистент |
| **R3** | Узгодження строків з клієнтом | B2B-менеджер |
| **R4** | Виконання робіт з усунення | Montageleiter |
| **R4.1** | Розрахунок фактичної собівартості | Ассистент бухгалтер |
| **R5** | Сповіщення клієнта і закриття | B2B-менеджер |

На етапі **R2** доступний розкривний **Акт діагностики рекламації** (клік по фразі
«заполняет Акт диагностики рекламации») — 11 полів із розділенням на попередні/фактичні
трудозатрати й матеріали.

## Файли

- `index.html` — дашборд (data-driven: весь контент у JS-об'єкті `data`, без бекенду)
- `B2B Wärmepumpe Montage — Рекламация.md` — канонічне джерело контенту воронки

## Як редагувати

Контент задається в JS-об'єкті `data` всередині `index.html`. Щоб змінити етап, роль,
чек-лист чи поле Акту — правте відповідний об'єкт; рендер оновиться автоматично.

## Технології

Чистий HTML/CSS/JS без залежностей. Адаптив під мобільні. Деплой — GitHub Pages.
